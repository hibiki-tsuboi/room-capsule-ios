# Codex CLI プロンプト：Room Capsule v4 全部入り iPhone アプリ

あなたはシニア iOS エンジニアです。現在の Xcode / Swift プロジェクトに、**Room Capsule** という iPhone アプリを実装してください。

このアプリは、収益化や業務向け機能ではなく、**「iPhone でこんなことできるんだ！すごい！」という AR / 空間スキャン体験**を最優先にします。

実装は v1 から v4 まで段階的に足すのではなく、最初から **v4 の完成形**を目指してください。ただし、デバイス・OS・API・依存ライブラリの制限で完全実装できない部分がある場合でも、アプリ全体がビルド可能で、体験の導線が崩れないように、代替実装・フォールバック・プレースホルダーを最小限入れてください。

---

## 0. 重要な進め方

- 既存の Xcode プロジェクトがある場合は、それを読み取って自然に統合してください。
- 既存プロジェクトがない場合は、SwiftUI ベースの iOS アプリとして必要なファイル構成を作成してください。
- 実装は **SwiftUI + ARKit + RealityKit + RoomPlan** を中心にしてください。
- Gaussian Splatting は、利用可能な環境に応じて以下の順で実装してください。
  1. 利用できる公式 RealityKit API がある場合はそれを使用。
  2. 使いやすい Swift / Metal ベースの Gaussian Splatting 表示ライブラリを導入できる場合は、それを使用。
  3. それが難しい場合でも、`.ply` / `.splat` / `.spz` ファイルのインポート、保存、ビューア画面、抽象化レイヤー、将来差し替え可能なレンダラーまでは必ず実装。
- どの機能も、未実装のボタンだけを置くのではなく、最低限触れる体験にしてください。
- ビルドが壊れるコード、存在しない API を決め打ちしたコード、コンパイル不能なサンプルコードは避けてください。
- 可能なら実機ビルドを想定し、シミュレータや LiDAR 非対応端末では丁寧なフォールバック画面を出してください。
- 実装後、README に使い方、必要な端末、制限事項、Gaussian Splatting 部分の状態を書いてください。

## 1. アプリのコンセプト

アプリ名：**Room Capsule**

一言でいうと、

> いま目の前にある部屋を iPhone でスキャンして、ミニチュア・実寸 AR・ポータル・写真のような 3D 空間として保存し、あとから再体験できるアプリ。

ユーザーに与えたい感覚は以下です。

- 部屋を iPhone の中に閉じ込めた。
- 自分の部屋が手のひらサイズのミニチュアになった。
- 昔の部屋を今いる場所に AR で呼び出せる。
- AR のドアから保存した部屋をのぞける。
- 部屋の Before / After を時間スライダーで見られる。
- 壁・家具・窓・ドアをタップすると、部屋が理解されている感じがする。

便利ツール感よりも、**空間をキャプチャして遊べる魔法感**を優先してください。

## 2. 実装したい v4 完成形

v4 では以下を全部入れてください。

### A. RoomPlan スキャン

- RoomPlan を使って部屋をスキャンできる。
- スキャン中に、部屋の認識が進んでいることが分かる UI を出す。
- スキャン完了後、以下を保存する。
  - 部屋名
  - 作成日時
  - RoomPlan の取得データ
  - 可能なら USDZ / USD などの 3D エクスポート
  - 壁、床、窓、ドア、開口部、家具などの構造情報
  - サムネイル
- RoomPlan / LiDAR 非対応端末では、サンプル部屋データで体験できるデモモードを用意する。

### B. ホーム画面 / 部屋カプセル一覧

- 保存した部屋をカード一覧で表示。
- 各カードには以下を表示。
  - 部屋名
  - サムネイルまたは簡易 3D アイコン
  - 作成日時
  - スキャン数 / バージョン数
  - メモ数
- 「部屋を保存する」ボタンから RoomPlan スキャンへ進む。
- 「デモ部屋を追加」ボタンも用意し、実機環境がなくてもアプリの体験を確認できるようにする。

### C. 部屋詳細画面

部屋を選ぶと、以下のモードへ入れる詳細画面を作る。

- ミニチュア AR
- 実寸 AR
- ポータル
- 写真っぽい 3D / Gaussian Splatting
- 間取り / 図面
- X 線 / 分解表示
- Before / After
- メモピン
- 家具ゴースト

UI は業務アプリではなく、楽しい実験アプリの雰囲気にしてください。

## 3. メイン体験：ミニチュア AR

最重要機能です。

### 体験

1. ユーザーが部屋をスキャンする。
2. スキャン結果が小さなドールハウスになる。
3. 現実の机や床の上に、その部屋を AR で置ける。
4. ピンチで拡大縮小できる。
5. ドラッグや回転ジェスチャで向きを変えられる。
6. 壁・窓・ドア・家具をタップすると情報が出る。
7. 表示モードを切り替えられる。

### 表示モード

- **模型モード**：白または淡い色のシンプルなドールハウス。
- **寸法モード**：壁の長さ、高さ、家具サイズなどを表示。
- **X 線モード**：壁を半透明にして中が見える。
- **家具だけモード**：家具・設備だけを浮かせる。
- **構造だけモード**：壁・床・窓・ドアだけを表示。
- **メモモード**：メモピンを浮かせる。
- **写真モード**：Gaussian Splatting または代替のテクスチャ / インポートデータを表示。

### 実装メモ

- RealityKit の Entity と Component を使って、RoomPlan の構造を 3D 表示してください。
- RoomPlan の詳細な変換が難しい場合でも、壁・床・窓・ドア・家具を簡易ジオメトリとして再構成してください。
- タップ判定、選択状態、インスペクタ表示を実装してください。
- 選択されたパーツはハイライトしてください。
- 触ったときに小さなアニメーションや haptics を入れてください。

---

## 4. 実寸 AR モード

保存した部屋を、今いる空間に 1:1 サイズで呼び出すモードです。

### 体験

- 「実寸で呼び出す」ボタンを押す。
- ユーザーが床をタップして原点を決める。
- 保存した部屋の壁・家具・窓・ドアが 1:1 スケールで表示される。
- 透明度スライダーで濃さを調整できる。
- 現実空間に昔の部屋や内見した部屋が重なって見える。

### 必須 UI

- 原点をリセット
- 透明度スライダー
- 表示モード切替
- ミニチュア / 実寸切替
- 選択した壁や家具の情報パネル

### フォールバック

AR が使えない環境では、3D プレビュー画面として表示してください。

---

## 5. ポータルモード

ARKit らしい「すごい」体験として、保存した部屋をのぞける AR ドアを実装してください。

### 体験

1. 現実の床に AR のドアフレームを置く。
2. ドアの向こう側に、保存した部屋の模型または Gaussian Splatting 空間が見える。
3. ドアをタップすると、部屋の中へ入ったような 3D ビューに遷移する。
4. 戻ると現実空間に戻る。

### 実装方針

- 完全なマスキング / クリッピングが難しい場合でも、以下のような「それっぽい」演出を作ってください。
  - RealityKit でドアフレームを作る。
  - フレーム内に部屋の縮小モデルまたは 3D ビューを配置する。
  - フレーム内を少し暗くして、奥行きがあるように見せる。
  - タップで「中へ入る」フルスクリーンビューに切り替える。
- 見た目のインパクトを優先してください。

## 6. Gaussian Splatting / 写真っぽい 3D モード

このモードは、白い RoomPlan 模型から一気に「本物っぽい部屋」に変わる驚きを作るためのものです。

### 必須機能

- `.ply` / `.splat` / `.spz` のファイルインポート導線を作る。
- 部屋カプセルに Gaussian Splatting ファイルを紐づけて保存できる。
- 部屋詳細画面から「写真っぽい 3D」モードを開ける。
- Gaussian Splatting の実レンダリングが可能なら、それを表示する。
- 実レンダリングが難しい場合でも、以下は必ず実装する。
  - ファイルインポート
  - ファイル管理
  - SplatViewer 画面
  - レンダラー抽象化プロトコル
  - 将来 Metal / RealityKit 実装に差し替えられる構造
  - ユーザーに「このビルドでは Splat 表示はプレビュー実装です」と分かる UI

### レンダラー設計

以下のような抽象化を作ってください。

- `SplatRenderable`
- `SplatRendererView`
- `SplatAsset`
- `SplatImportService`
- `SplatRendererAvailability`

外部ライブラリを導入する場合でも、アプリ本体がそのライブラリに密結合しないようにしてください。

### 体験演出

- 部屋詳細で「写真モード」を押す。
- 模型がフェードアウトし、写真っぽい空間がフェードインする。
- まだ Splat データがない場合は「Splat を追加」画面を出す。
- サンプル Splat を入れられるなら入れてください。難しければデモ用の美しいプレースホルダーを作ってください。

## 7. Before / After タイムライン

同じ部屋を複数回スキャンして、時間で比較できるようにしてください。

### 体験

- 部屋に複数の「バージョン」を保存できる。
  - 入居前
  - 家具配置後
  - リフォーム前
  - リフォーム後
  - 退去前
- Before と After を選ぶ。
- スライダーを動かすと、部屋表示が Before から After に切り替わる。

### 実装

- 同じ RoomCapsule に複数の RoomScanVersion を持たせる。
- 2 つのバージョンを選択できる UI を作る。
- スライダーで以下を補間または切り替え表示する。
  - 模型の透明度クロスフェード
  - 家具の表示 / 非表示
  - メモピンの表示切替
  - Gaussian Splatting がある場合は Splat 表示の切替またはクロスフェード
- 完全な形状補間が難しい場合は、透明度クロスフェードで十分です。

## 8. メモピン

### 機能

- 3D 空間上にメモピンを置ける。
- メモピンには以下を保存できる。
  - タイトル
  - 本文
  - カテゴリ
  - 作成日時
  - 3D 座標
  - 紐づく部屋バージョン
  - 写真 1 枚以上、可能なら対応
- ピンをタップすると詳細カードが出る。
- ピンのカテゴリ例：
  - 傷
  - お気に入り
  - 家具候補
  - リフォーム案
  - 内見メモ
  - 思い出

### 体験

- 部屋の中に小さな光るピンが浮いている。
- タップするとメモがふわっと出る。
- メモピンはミニチュア AR、実寸 AR、3D プレビューで見られる。

## 11. データ設計

最低限、以下のようなモデルを作ってください。名前はプロジェクトに合わせて自然に調整して構いません。

```swift
struct RoomCapsule: Identifiable, Codable {
    var id: UUID
    var name: String
    var createdAt: Date
    var updatedAt: Date
    var versions: [RoomScanVersion]
    var memoPins: [RoomMemoPin]
    var furnitureGhosts: [FurnitureGhost]
}

struct RoomScanVersion: Identifiable, Codable {
    var id: UUID
    var name: String
    var capturedAt: Date
    var roomDataURL: URL?
    var usdzURL: URL?
    var thumbnailURL: URL?
    var splatAsset: SplatAsset?
    var simplifiedGeometry: SimplifiedRoomGeometry
}

struct SimplifiedRoomGeometry: Codable {
    var walls: [RoomWall]
    var openings: [RoomOpening]
    var furniture: [RoomFurniture]
    var floor: RoomFloor?
}

struct RoomMemoPin: Identifiable, Codable {
    var id: UUID
    var title: String
    var body: String
    var category: MemoCategory
    var createdAt: Date
    var position: SIMD3<Float>
    var versionID: UUID?
    var photoURLs: [URL]
}

struct FurnitureGhost: Identifiable, Codable {
    var id: UUID
    var type: FurnitureGhostType
    var name: String
    var position: SIMD3<Float>
    var rotationY: Float
    var size: SIMD3<Float>
    var versionID: UUID?
}

struct SplatAsset: Identifiable, Codable {
    var id: UUID
    var fileURL: URL
    var fileType: SplatFileType
    var importedAt: Date
}
```

`SIMD3<Float>` はそのまま Codable にできない場合があるため、必要なら Codable 対応のラッパー型を作ってください。

永続化は以下のどちらかで実装してください。

- SwiftData
- JSON ファイル + Documents ディレクトリ保存

シンプルで壊れにくい方を選んでください。

## 13. RealityKit / AR 実装方針

- AR 表示は `ARView` を SwiftUI にラップして使ってください。
- 平面検出を使い、ユーザーがタップした場所に部屋モデルを置けるようにしてください。
- RoomPlan 由来の部屋モデルは、まず簡易ジオメトリで確実に表示してください。
- 壁は薄い Box、床は Plane / Box、家具は Box またはプリミティブで表現してください。
- 窓やドアは壁上のラベルや薄い枠として表現して構いません。
- 可能なら USDZ エクスポートを読み込んで表示してください。ただし、USDZ 読み込みだけに依存せず、簡易ジオメトリ表示でも成立するようにしてください。
- Entity 生成は `RoomEntityFactory` のような専用クラスに分離してください。

## 15. プライバシー / ローカル保存

- 収益化やクラウド同期は実装しないでください。

## 18. 受け入れ条件

実装完了時点で、以下を満たしてください。

1. アプリがビルドできる。
2. ホーム画面が表示される。
3. デモ部屋を追加できる。
4. 部屋詳細から各モードに移動できる。
5. RoomPlan 対応端末では部屋スキャンを開始できる。
6. RoomPlan 非対応端末ではデモモードへ誘導される。

## 21. 絶対に避けること

- 収益化機能を入れない。
- ログイン必須にしない。
- クラウド前提にしない。
- 存在しない API を決め打ちしない。
- コンパイルできないコードを残さない。
- 空のボタンだけを大量に置かない。
- サンプルデータなしで実機がないと何も見られない状態にしない。
- Gaussian Splatting が難しいからといって、導線や設計まで消さない。

## 22. 目指す完成イメージ

最終的に、ユーザーがこう体験できるアプリにしてください。

1. アプリを開く。
2. 「部屋を保存する」を押す。
3. iPhone で部屋をぐるっとスキャンする。
4. 部屋が小さなカプセルとして保存される。
5. 机の上に自分の部屋のミニチュアを置く。
6. 壁や家具をタップして、iPhone が部屋を理解していることを感じる。
7. 実寸モードで、昔の部屋を今いる空間に呼び出す。
8. ポータルモードで、AR のドアから保存した部屋をのぞく。
9. Before / After スライダーで部屋の時間を戻す。
10. Gaussian Splatting がある場合は、白い模型が写真のような空間に変わる。

この「部屋を保存して、呼び出して、のぞいて、時間を戻せる」体験を最優先に、Room Capsule を実装してください。
